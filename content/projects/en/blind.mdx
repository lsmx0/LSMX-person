## Core Features

### 1. Real-time Blind Path Detection
- **YOLOv8-Seg Model Inference**: Runs the YOLOv8-Seg model on-device using ONNX Runtime.
- **Instance Segmentation**: Accurately identifies blind path areas and generates pixel-level segmentation masks.
- **Real-time Processing**: Achieves ~6-7 FPS inference speed, meeting real-time navigation requirements.

### 2. Intelligent Navigation Algorithm
- **Centerline Extraction**: Extracts the geometric centerline of the blind path from the segmentation mask.
- **Linear Fitting**: Uses least squares to fit the centerline and calculate the path direction.
- **Turn Decision**: Determines left turn, right turn, or straight based on the centerline angle.
- **Offset Detection**: Detects whether the user has deviated from the center of the blind path.

### 3. Voice Navigation
- **Real-time Voice Broadcast**: Provides Chinese voice guidance using Android TTS.
- **Smart Rate Limiting**: Does not repeat the same content within 2 seconds to avoid disturbance.
- **Priority Decision**: Turn prompts take priority over offset prompts.

### 4. Visual Debugging
- **Real-time Overlay**: Displays detection results overlaid on the camera preview.
- **Segmentation Mask Display**: Shows identified blind path area as a semi-transparent green overlay.
- **Centroid Marker**: Red dot marking the centroid of the blind path.
- **Center Reference Line**: White dashed line showing the center of the screen.
- **Direction Arrow**: Yellow arrow indicating the direction of the blind path.
- **Status Text**: Real-time display of offset and turn status.

### 5. Dual Mode Support
- **Image Detection Mode**: For debugging and testing, select an image for detection.
- **Real-time Detection Mode**: Main function, real-time camera detection and navigation.

### Core Components

#### 1. YoloSegModel.kt
**Function**: YOLOv8-Seg model inference engine

**Key Implementation**:
```kotlin
// Model Loading
- Load best.onnx model using ONNX Runtime
- Configure optimization level to ALL_OPT

// Image Preprocessing
- Bitmap → FloatBuffer
- RGB format, CHW layout
- Normalize to [0, 1]
- Resize to 640×640

// Inference Execution
- Input: [1, 3, 640, 640] float32 tensor
- Outputs:
  * det: [1, 8400, 37] - detection results (bbox + conf + mask_coeff)
  * proto: [1, 32, 160, 160] - mask prototypes

// Mask Generation
- Formula: mask = sigmoid(mask_coeff × proto)
- Bbox Cropping: Crop 160×160 mask to detection box region
- Output: 640×640 segmentation mask
```

**Output Format Parsing**:
- Each detection: 37 features
  - [0-3]: bbox coordinates (cx, cy, w, h)
  - [4]: confidence
  - [5-36]: 32-dimensional mask coefficients

#### 2. CameraManager.kt
**Function**: Camera management and image capture

**Key Implementation**:
```kotlin
// CameraX Configuration
- Preview: Real-time preview
- ImageAnalysis: Image analysis (inference)
- Target resolution: 1280×720
- Target aspect ratio: 4:3

// Performance Control
- Single-thread inference lock: avoid frame stacking
- Time interval limit: one inference per 150ms (~6-7 FPS)
- Backpressure strategy: KEEP_ONLY_LATEST

// YUV to Bitmap Conversion (Critical Fix)
- Correctly handle stride and pixel stride of YUV planes
- Pixel-by-pixel copy to avoid data misalignment
- Support image rotation correction
```

**YUV Conversion Fix**:
The original incorrect method caused images to appear as horizontal stripes. After the fix, each pixel position is correctly handled:
```kotlin
// Incorrect method (direct buffer copy)
yBuffer.get(nv21, 0, ySize)  // ❌

// Correct method (pixel-by-pixel copy with stride)
for (row in 0 until height) {
    for (col in 0 until width) {
        nv21[pos++] = yBuffer.get(row * yRowStride + col * yPixelStride)  // ✅
    }
}
```

#### 3. BlindPathGuide.kt
**Function**: Blind path navigation algorithm and voice broadcast

**Core Algorithms**:

##### 3.1 Centerline Extraction
```kotlin
Method:
1. Segment the image into rows (every 20 rows)
2. Scan each row to find the left and right boundaries of foreground pixels
3. Calculate center point: centerX = (minX + maxX) / 2
4. Obtain list of centerline points: [(x1, y1), (x2, y2), ...]
```

##### 3.2 Linear Fitting
```kotlin
Method: Least squares fitting of line x = a * y + b
- a: slope (indicates tilt of the blind path)
- b: intercept (indicates position of the blind path)

Formula:
denominator = n * sumYY - sumY * sumY
a = (n * sumXY - sumX * sumY) / denominator
b = (sumX * sumYY - sumY * sumXY) / denominator
```

##### 3.3 Turn Decision
```kotlin
Method: Calculate angle from slope
angle = atan(slope) * 180 / π

Decision Rules:
- angle > 10°  → turn left
- angle < -10° → turn right
- otherwise    → go straight

Angle Interpretation:
- 0° = vertical upward (straight)
- Positive = tilting right (turn left)
- Negative = tilting left (turn right)
```

##### 3.4 Offset Detection
```kotlin
Method: Calculate centerline position at the middle of the frame
midY = imageHeight / 2
midX = slope * midY + intercept
offset = midX - centerX

Decision Rules:
- offset < -15% of frame width → left deviation
- offset > +15% of frame width → right deviation
- otherwise                     → centered
```

##### 3.5 Decision Fusion
```kotlin
Priority:
1. Turn prompt (turn left / turn right)
2. Offset prompt (left deviation / right deviation)
3. No prompt (maintain current direction)

Voice Mapping:
- turn left → "Turn left ahead"
- turn right → "Turn right ahead"
- left deviation → "Adjust right"
- right deviation → "Adjust left"
```

##### 3.6 Voice Broadcast
```kotlin
Features:
- Uses Android TTS (Chinese)
- Rate limiting: same content not repeated within 2 seconds
- Queue mode: QUEUE_FLUSH (immediate broadcast)
```

## 🚀 Usage Instructions

### Requirements
- Android 7.0 (API 24) or higher
- Camera permission
- Chinese TTS voice pack (optional)

### Installation Steps
1. Clone the project
2. Open with Android Studio
3. Place the trained `best.onnx` model into `app/src/main/assets/`
4. Build and install on device

### Usage Flow

#### Image Detection Mode (Debug)
1. Switch to the "Image Detection" tab
2. Click "Select Image"
3. Adjust confidence threshold (recommended 0.5)
4. Click "Run Segmentation"
5. View segmentation results and navigation info

#### Real-time Detection Mode (Main Function)
1. Switch to the "Real-time Detection" tab
2. Grant camera permission
3. Click "Start Detection"
4. Point the camera at the blind path
5. Listen to voice navigation guidance
6. Optional: Enable "Show Overlay" to view the detection process

### Debugging Tips

#### View Logs
Use Logcat to filter the following tags:
```
🎯 Centerline Analysis  - Centerline extraction and fitting results
🔊 Voice Broadcast      - TTS status and broadcast information
🔄 YUV Conversion       - Image conversion process
✅/❌                  - Success/failure status
```

#### Common Issues

**1. No target detected**
- Lower confidence threshold (0.5 → 0.3)
- Ensure sufficient lighting
- Aim clearly at the blind path
- Keep distance of 1-2 meters

**2. No voice broadcast**
- Check TTS initialization logs in Logcat
- Verify Chinese voice pack is installed on device
- Enable TTS function in settings

**3. Abnormal image display**
- Check if YUV conversion is correct
- View saved camera frames
- Confirm image rotation angle

**4. Inaccurate turn decision**
- Check centerline analysis logs in Logcat
- Examine fitted slope and angle values
- Adjust `turnAngleThreshold` parameter

## 🎯 Core Algorithm Advantages

### Centerline Method vs PCA Method

**Centerline Method** (currently used):
- ✅ More accurate: directly extracts geometric features of the blind path
- ✅ More stable: linear fitting smooths noise
- ✅ More intuitive: clear meaning of angle and offset
- ✅ More robust: has fallback mechanism

**PCA Method** (deprecated):
- ❌ Sensitive to noise
- ❌ Ambiguous angle interpretation
- ❌ Prone to misclassifying straight as turn
